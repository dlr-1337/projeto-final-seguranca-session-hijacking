# Plan 01-01 - Create Express App Shell

<objective>
As a team member, I want to start a local Express app and navigate through a fake login/dashboard shell, so that the vulnerable lab has a working classroom foundation before session behavior is added.

Purpose: Use TDD to lock the observable local app behavior before implementation.
Output: A runnable Express app with documented commands, fake users, login page, dashboard page, and route tests.
</objective>

| ID | Threat | Severity | Mitigation In This Plan |
|----|--------|----------|-------------------------|
| T-01-01 | Real credentials or private data are accidentally introduced into the demo. | high | Use only hard-coded fictitious users and fake dashboard data from `src/data/users.js`; README must state data is fictional. |

| T-01-03 | UI distracts from session behavior. | low | Follow `01-UI-SPEC.md`; keep login/dashboard compact and utilitarian. |
