# Plan 01-02 - Implement Vulnerable Session Behavior

<objective>
As a demo operator, I want valid login to create an intentionally vulnerable `sid` session and protect the dashboard, so that Phase 2 can prove cookie reuse in a local, controlled lab.

Purpose: Use TDD to lock both the authentication behavior and the intentionally insecure cookie contract.
Output: Vulnerable session middleware, protected dashboard access, tests for cookie attributes, and README evidence.
</objective>

Update `POST /login` to validate fake credentials and set `req.session.userId`. Add `src/middleware/require-auth.js` to protect `GET /dashboard`; unauthenticated requests should redirect to `/login`. Dashboard should read the session user and render that user's fake private data. Keep default host `127.0.0.1` and port `3000`.
