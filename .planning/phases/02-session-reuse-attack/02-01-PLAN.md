# Plan 02-01: Criar roteiro de ataque com navegador, DevTools e cURL

<objective>
As a demo presenter, I want a clear local attack walkthrough using browser DevTools and a cURL backup so that the team can rehearse and demonstrate session cookie reuse without external targets.

This plan covers VULN-02 and ATK-01 by documenting how to copy the vulnerable `sid` cookie and replay it in a second local client. It produces the manual script that Plan 02-02 will verify with evidence and an automated replay proof.
</objective>

<must_haves>
- VULN-02: The documentation must show that the vulnerable session cookie can be reused by a second local client.
- ATK-01: The documentation must include a reproducible, controlled local cookie-copy path.
- D-01: Browser plus DevTools is the primary demonstration path.
- D-02: cURL is the required reproducible backup path; Postman can be optional only.
- D-03: The second client must be shown denied before replay.
- D-04: The walkthrough copies only the `sid` value or sends `Cookie: sid=<value>`.
- D-05: The plan must not add a helper endpoint, browser storage export, packet sniffing, XSS, or broader attack tooling.
- D-06: The explanation must connect cookie replay to server-side sessions.
- D-10: All instructions stay local on `127.0.0.1`/localhost with fake users.
- D-11: Reset/cleanup steps must be included.
- D-12: The violated principle must be explained in plain terms.
- D-13: Create a focused walkthrough document, preferably `docs/session-reuse-attack.md`.
- D-14: Update `README.md` with a Phase 2 link or short section.
- D-15: Keep changes minimal and do not implement fixed-mode session behavior.
</must_haves>
