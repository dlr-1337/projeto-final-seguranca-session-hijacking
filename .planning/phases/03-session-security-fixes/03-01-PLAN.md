# Plan 03-01 - Add Fixed Session Mode

<objective>
As a demo operator, I want to start the same local app in vulnerable mode or fixed mode so that the team can compare insecure and corrected cookie settings without editing source code.

Purpose: Use TDD to lock the corrected cookie contract before implementation.
Output: Mode-aware session setup, fixed cookie helper, fixed-mode test coverage, and README commands for vulnerable/fixed startup.
</objective>

<threat_model>

| ID | Threat | Severity | Mitigation In This Plan |
|----|--------|----------|-------------------------|
| T-03-01 | Fixed mode accidentally replaces vulnerable mode and breaks the before/after attack demo. | high | Keep vulnerable mode as an explicit mode and keep existing vulnerable tests passing. |
| T-03-02 | Secure cookie behavior is misrepresented while running over plain HTTP. | high | Test the secure fixed cookie path and document any HTTP fallback as local inspection/testing only. |
| T-03-03 | Cookie prefix rules are implemented incorrectly. | medium | Use `__Host-session` only with Secure, Path=/, and no Domain; use a clearly different fallback name if Secure is disabled. |
| T-03-04 | Session lifetime is too long to support the mitigation story. | medium | Add short fixed maxAge, preferably 5 minutes, and assert expiration metadata in tests. |

</threat_model>

<must_haves>
- FIX-01: The app has a corrected mode with `HttpOnly`, `Secure`, `SameSite`, and appropriate expiration.
- D-01: Preserve vulnerable mode and add explicit fixed mode selection.
- D-02: `createApp()` accepts optional mode/config for tests while server startup reads env vars.
- D-03: Running vulnerable and fixed modes does not require source edits.
- D-04: Fixed settings live in a named helper or session factory.
- D-05: Fixed cookie policy includes `httpOnly: true`, `secure: true`, `sameSite: "strict"`, and short `maxAge`.
- D-06: `__Host-session` is used only for the secure fixed path.
- D-07: Fixed cookie uses Path=/ and no Domain.
- D-08: Any HTTP fallback is labeled as local inspection/testing only.
- D-15: Existing vulnerable tests remain intact and focused fixed cookie tests are added.
</must_haves>
